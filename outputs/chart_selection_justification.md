# Chart Selection Justification
## Retail Executive Dashboard — Sales & Business Intelligence (2024–2025)

---

## Chart 1: Sales Trend View — Dual Axis Line Chart

**Question the chart answers:**
How are monthly sales and profit changing over time across 2024–2025?

**Why this chart type is appropriate:**
A line chart is the standard and most effective chart for showing continuous change over time. The dual axis configuration allows Sales and Profit to be displayed on the same time axis with independent scales, enabling direct visual comparison of their trends without one dominating the other. A bar chart would have worked but would be harder to read at 24 monthly data points.

**Fields used:**
- X-axis (Columns): MONTH(Order Date) — continuous date, green pill
- Y-axis left (Rows): SUM(Sales) — blue line
- Y-axis right (Rows): SUM(Profit) — orange line, dual axis
- Filter: YEAR(Order Date) to restrict to 2024–2025
- Color: Measure Names (auto-assigned blue for Sales, orange for Profit)

**Design principle applied:**
Dual axis with synchronized scale labels on both sides. Removed mark labels to reduce clutter — values are accessible via tooltip on hover. Legend placed on right side for quick reference.

**Mistake avoided:**
Did not use a bar chart at monthly level — 24 bars would create visual noise and make trend direction harder to read. Did not use a stacked area chart — that would imply Sales and Profit are additive components of the same whole, which is misleading.

## Chart 2: Regional Performance View — Filled Map (India States)

**Question the chart answers:**
Which Indian states generate the highest sales, and how does performance vary geographically?

**Why this chart type is appropriate:**
A geographic map is the only chart type that communicates spatial distribution intuitively. When the data has a geographic dimension (19 Indian states), a map allows the leadership team to instantly identify high and low performing regions without reading a table. Color intensity (Sales) provides immediate visual ranking.

**Fields used:**
- Columns: Longitude (generated by Tableau)
- Rows: Latitude (generated by Tableau)
- Mark type: Map (Filled)
- Color: SUM(Sales) — sequential blue (darker = higher sales)
- Detail: State (geographic role assigned)
- Detail: Region (for tooltip context)
- Tooltip: SUM(Profit)

**Design principle applied:**
Single sequential color palette (blue) used for Sales to indicate magnitude. States with no data appear in grey, making coverage gaps immediately visible. Map title clearly states the purpose.

**Mistake avoided:**
Did not use a bubble map over a filled map — filled map better shows state boundaries and is easier to read for India's varied state sizes. Did not use Region as the geographic unit — state-level granularity provides more actionable insight for field teams.

## Chart 3: Regional Bar View — Horizontal Bar Chart

**Question the chart answers:**
Which region has the highest total sales and how do they rank against each other?

**Why this chart type is appropriate:**
A horizontal bar chart is ideal for comparing a single measure (Sales) across a small number of categories (4 regions). The horizontal orientation allows region names to be read easily without rotation. Color by SUM(Profit) adds a second dimension, enabling the viewer to simultaneously assess sales volume and profitability.

**Fields used:**
- Columns: SUM(Sales)
- Rows: Region
- Color: SUM(Profit) — diverging color (higher profit = darker blue)
- Sorted: Descending by SUM(Sales)

**Design principle applied:**
Sorted descending so the highest performing region appears at the top — consistent with how leadership reads ranked lists. Color adds profit context without requiring a second chart.

**Mistake avoided:**
Did not use a pie chart — pie charts are ineffective for comparing similar-sized values (all four regions are within 25% of each other) and make ranking impossible. Did not use a vertical bar — horizontal bars are easier to read when category labels are text strings.

## Chart 4: Category Profitability View — Horizontal Bar Chart with Hierarchy

**Question the chart answers:**
Which product categories and sub-categories drive the most profit, and which are underperforming?

**Why this chart type is appropriate:**
A horizontal bar chart with a Category → Sub Category row hierarchy clearly shows both the category-level and sub-category-level profit in a single view. The hierarchical rows allow drill-down without requiring separate charts. A treemap was considered but rejected because it makes precise value comparison difficult.

**Fields used:**
- Columns: SUM(Profit)
- Rows: Category → Sub Category (hierarchy)
- Color: SUM(Profit) — diverging blue (higher = darker)
- Label: SUM(Sales) — shows sales context alongside profit bars
- Tooltip: Profit Margin calculated field

**Design principle applied:**
Category grouping via row hierarchy creates visual separation between Furniture, Office Supplies, and Technology without needing color coding by category. Labels show absolute profit values for precise reading. Sorted by profit descending within each category.

**Mistake avoided:**
Did not use a stacked bar by category — that would hide sub-category level detail. Did not use negative diverging colors since all profit values in this dataset are positive at the sub-category aggregate level.

## Chart 5: Customer Segment View — Grouped Bar Chart

**Question the chart answers:**
How do Consumer, Corporate, and Home Office segments compare on Sales and Profit?

**Why this chart type is appropriate:**
A grouped bar chart allows direct side-by-side comparison of two measures (Sales and Profit) across three customer segments simultaneously. This is more efficient than two separate bar charts and immediately shows both the absolute values and the relative proportion of profit to sales per segment.

**Fields used:**
- Columns: Customer Segment, Measure Names
- Rows: Measure Values
- Marks: Bar
- Color: Measure Names (Sales = blue, Profit = orange)
- Label: Measure Values (shows exact numbers above bars)
- Tooltip: AGG(Average Order Value), AGG(Return Rate)

**Design principle applied:**
Consistent color assignment — blue for Sales and orange for Profit — matches the color scheme used in the Sales Trend View, creating visual consistency across the dashboard. Labels on bars allow precise reading without requiring the viewer to reference the axis.

**Mistake avoided:**
Did not use a line chart — segments are discrete categories, not a time series, so connecting them with a line would imply a continuous relationship that does not exist. Did not use a stacked bar — that would imply Sales and Profit are parts of a whole, which is misleading.

## Chart 6: Shipping Performance View — Highlight Table (Heatmap)

**Question the chart answers:**
Which shipping modes experience the most delays, and how many orders fall into each delivery speed category?

**Why this chart type is appropriate:**
A highlight table (heatmap with square marks) is the most efficient way to display a matrix of two categorical variables (Ship Mode × Shipping Delay Bucket) with a quantitative measure (order count). The color intensity immediately draws the eye to high-volume combinations, and the numbers in each cell provide precise counts. A bar chart would require either grouping or faceting, making comparison across both dimensions harder.

**Fields used:**
- Columns: Shipping Delay Bucket (calculated field)
- Rows: Ship Mode
- Mark type: Square
- Color: CNT(Order Id) — sequential blue (darker = more orders)
- Label: CNT(Order Id) — exact count in each cell

**Design principle applied:**
Sequential single-color palette where intensity encodes volume. Empty cells (white) visually confirm logical impossibilities — e.g., Same Day shipping cannot result in Slow (5-6 day) delivery. Column order arranged logically: Same Day → Fast → Standard → Slow → Very Slow.

**Mistake avoided:**
Did not use a treemap — while Tableau auto-suggested it, a treemap cannot show the matrix relationship between two categorical dimensions. Did not use a stacked bar — a heatmap is far more readable for this type of cross-tabulation.

## Chart 7: Discount vs Profit View — Scatter Plot

**Question the chart answers:**
What is the relationship between discount rate and profit, and does this relationship differ by product category?

**Why this chart type is appropriate:**
A scatter plot is the only chart type specifically designed to show the relationship (correlation) between two continuous numerical variables. With Discount on the X-axis and Profit on the Y-axis, the trend line immediately reveals the negative relationship. No other chart type can show this pattern as clearly. Color by Category adds a third dimension, revealing that the relationship is steepest for Furniture.

**Fields used:**
- Columns: Discount (continuous measure)
- Rows: SUM(Profit)
- Mark type: Circle
- Color: Category (3 distinct colors — Furniture, Office Supplies, Technology)
- Size: SUM(Sales) — larger dots = higher sales value
- Detail: Sub Category — each dot represents one sub-category at one discount level
- Trend Lines: One per category (linear)

**Design principle applied:**
Three distinct colors for three categories enable immediate group identification. Trend lines per category allow comparison of discount sensitivity across categories. Size encoding adds sales context without requiring a fourth axis.

**Mistake avoided:**
Did not use a bar chart — a bar chart cannot show correlation between two continuous variables. Did not use a single trend line for all categories — that would hide the fact that Furniture is far more sensitive to discounts than Technology.

## Chart 8: Return Analysis View — Stacked Horizontal Bar Chart

**Question the chart answers:**
Which product categories and customer segments have the highest return rates, and how do regions contribute to returns?

**Why this chart type is appropriate:**
A stacked horizontal bar chart shows both the total return rate per Category-Segment combination (bar length) and the regional breakdown within each bar (stacked color segments). This provides three dimensions of information — Category, Customer Segment, and Region — in a single readable chart. A simple bar would lose the regional dimension; a separate chart would require more dashboard space.

**Fields used:**
- Columns: AGG(Return Rate Avg) — average return rate
- Rows: Category → Customer Segment (hierarchy)
- Color: Region (4 colors — East, North, South, West)
- Label: SUM(Return Flag) — shows actual return count per segment
- Mark type: Bar (automatic horizontal)

**Design principle applied:**
Row hierarchy (Category → Segment) creates visual grouping that makes category-level comparison easy while preserving segment detail. Return counts as labels provide absolute numbers alongside the percentage-based axis. Color by Region reveals geographic concentration of returns.

**Mistake avoided:**
Did not keep all three dimensions (Category, Segment, Region) as separate rows — that created 36 bars and was unreadable. Instead Region was moved to Color, reducing to 9 bars while preserving regional insight. Did not use a pie chart — return rates across segments are too similar in magnitude for pie to show meaningful differences.
